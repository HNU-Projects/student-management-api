# Student Management System - Frontend Development Guide

## 1. Project Overview
This is a Student Management System API built with FastAPI. It handles student records, authentication, and system monitoring.

## 2. Backend Technical Details
- **Base URL:** http://localhost:8000
- **Auth Type:** JWT Bearer Token
- **Key Entities:**
  - **User:** (Email, Password, Role [admin/student])
  - **Student:** (University ID, Name, GPA, Department, Status, etc.)

## 3. Main Endpoints for Frontend
### Authentication
- `POST /auth/register`: Create a new user.
- `POST /auth/login`: Returns `access_token` and `token_type`.
- `GET /users/me`: Get current logged-in user info.

### Students (Admin Only for Write, Student for Read own)
- `GET /students/`: List students (Support pagination & filters: `department`, `min_gpa`, `max_gpa`).
- `POST /students/`: Create a student record.
- `GET /students/{id}`: Get specific student details.
- `PUT /students/{id}`: Update student info.
- `DELETE /students/{id}`: Remove a student.
- `GET /students/stats/summary`: Statistical summary for dashboard cards.

### Monitoring
- `GET /monitoring/metrics`: System health and traffic stats.
- `GET /monitoring/dashboard`: HTML dashboard (already built in backend).
